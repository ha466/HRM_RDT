🔹 Data Flow Pipeline

Input Tokens → Embedding → Prelude → Recurrent Block [H ↔ L ×K] → Coda → RMSNorm → LM Head → Output

The architecture replaces standard feed-forward depth with temporal recurrence. Instead of stacking dozens of transformer layers, HRM-RDT loops a compact hierarchical pair, trading spatial depth for controlled computational time.

🧠 How HRM Works

HRM introduces four learnable components that operate on distinct timescales:

  • Input Network: Encodes discrete tokens into continuous latent vectors.
  • Low-Level L Module: Executes fast, granular computation. Loops until it reaches a local equilibrium, then resets.
  • High-Level H Module: Performs slow, abstract reasoning. Advances only after L converges, issuing a new guidance state zH.
  • Output Network: Projects the final H hidden state into token logits or task predictions.

Hierarchical Convergence

The critical insight is timescale separation. L runs aggressively until its hidden state stabilizes (local equilibrium). At that boundary, H takes a single deliberate step, updates zH, and L resets to begin a new phase conditioned on the fresh abstract context. This mimics cognitive fast/slow thinking loops while keeping gradients tractable.

⚙️ Module Breakdown

🔹 H Module (High-Level)

Slow, abstract reasoning. Takes one step per full L-convergence cycle. Its latent vector zH acts as a dynamic prior that steers the next inner L cycle via cross-attention.

🔸 L Module (Low-Level)

Fast, detailed computation. Loops until local equilibrium, then resets. Conditioned on H's current hidden state at every step, enabling fine-grained token manipulation without losing global context.

🔗 Coupling Mechanism

Cross-attention bridges H and L without hard dependency. H broadcasts abstract constraints; L resolves them into concrete representations. RMSNorm between iterations prevents activation drift.

🌌 OpenMythos × HRM Fusion

How OpenMythos Works

OpenMythos introduces a recurrent transformer block with the update rule:
h_{t+1} = A·h_t + B·e + Transformer(h_t, e)
The encoded input e (from the Prelude) is re-injected at every loop step. Without this, deep recurrence causes signal drift. Learned matrices A and B control state retention vs. input refresh. Crucially, the spectral radius of A is constrained to < 1, guaranteeing bounded dynamics regardless of learning rate or gradient noise.

The Hybrid Architecture

The fusion is conceptually clean: OpenMythos's single TransformerBlock inside the Recurrent Block is replaced by HRM's H+L pair. Instead of one block looping T times, you get:

  • L looping T_L times per step until convergence
  • H taking one slow step
  • The entire structure looping T_outer times
  • e re-injected at every inner L step for LTI stability

The OpenMythos stability guarantee applies to the outer recurrence, keeping the nested hierarchy bounded across arbitrary computational depth. This yields adaptive compute: easy inputs exit early, complex reasoning triggers deeper L/H cycles.

📊 Dataset & Training Dynamics

Dataset Specification

  • Size: 1,000 samples
  • Domain: Human-based reasoning traces
  • Format: JSON { input, output, reasoning_trace }
  • Split: 800 train / 100 val / 100 test
  • Stack: Lightning.ai · PyTorch + HuggingFace

Training Behavior

Due to the contractive outer recurrence and RMSNorm gating, gradients remain well-conditioned even with unrolled depth. Early-exit thresholds are learned dynamically, allowing the model to allocate compute proportionally to reasoning difficulty. The small parameter count forces the architecture to rely on temporal reuse rather than spatial width.
